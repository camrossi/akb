• A working ESXi infrastructure connected to ACI
  • A port-group to carry the traffic between the K8s VMs and the ACI leaves. This must be manually created in vCenter. There is no requirement for vCenter to be integrated via a VMM domain
  • A folder to store the K8s VMs, sub-folders are not supported at the moment
  • A data store, shared if the VMs are required to run on more than one host
  • Ubuntu VM template with the VMWare provisioning tool installed. For simplicity a VM Template is provided with the VMWare provisioning tool installed as well as all the packages and container images. This speed up the installation process (from ~30min to 5min) and does not require your K8s cluster to have internet connectivity. This is great for labs/sandbox environments
